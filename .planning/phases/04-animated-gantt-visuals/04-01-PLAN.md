**Component architecture:**
- AnimatedGantt is a client component (`"use client"` directive at top)
- Uses `useAnimate` from `"motion/react"` for choreographed timeline
- Uses `useReducedMotion` from `"motion/react"` for accessibility
- The `scope` ref from `useAnimate` goes on the `<svg>` element
- Each bar gets a CSS class `bar-{index}` for targeting by the animate function

**Layout constants (same as existing GanttMockup.tsx):**
```typescript

const CHART_LEFT = LABEL_WIDTH;
const CHART_WIDTH = 800 - LABEL_WIDTH; // 710
const DAY_WIDTH = CHART_WIDTH / 5; // 142
const HEADER_HEIGHT = 40;
const ROW_HEIGHT = 60;
const BAR_PADDING = 14;
const BAR_HEIGHT = ROW_HEIGHT - 28; // 32
```

**Helper functions:**
```typescript
function barX(startDay: number) { return CHART_LEFT + startDay * DAY_WIDTH; }
function barW(startDay: number, endDay: number) { return (endDay - startDay) * DAY_WIDTH; }
function barY(row: number) { return HEADER_HEIGHT + row * ROW_HEIGHT + BAR_PADDING; }
```

Test oklch first. If it works, keep it. If it snaps, switch to hex.

**Animation choreography (in `useEffect`):**
1. Bars render initially in chaos positions with `opacity: 0.7` (semi-transparent)
2. Hold for 1 second (the `at` parameter on the first bar starts at 1.0)
3. Each bar animates to its optimized position over 0.8s with easeInOut easing
4. Bars are staggered by 80ms (`at: 1 + i * 0.08` for bar index i)
5. Final opacity is 0.9

The animation sequence built for `animate()`:
```typescript
const sequence: AnimationSequence = [];
OPTIMIZED_BARS.forEach((optimized, i) => {
  const [row, startDay, endDay, fill] = optimized;
  sequence.push([
    `.bar-${i}`,
    {
      x: barX(startDay),
      width: barW(startDay, endDay),
      y: barY(row),
      fill: fill,
      opacity: 0.9,
    },
    {
      duration: 0.8,
      ease: [0.25, 0.1, 0.25, 1],
      at: 1 + i * 0.08,
    },
  ]);
});
await animate(sequence);
```

**IMPORTANT: SVG attribute animation.** The `useAnimate` hook's `animate()` function targets elements by CSS selector within the scoped ref. For SVG `<rect>` elements, `x`, `y`, `width`, and `fill` should be passed directly. If Motion interprets `x`/`y` as CSS transforms instead of SVG attributes (bars move from origin instead of from their chaos position), try these fallbacks in order:
1. Use `attrX` and `attrY` instead of `x` and `y`
2. If that doesn't work, switch from `useAnimate` to rendering `<motion.rect>` components with `initial` and `animate` props that set the SVG attributes explicitly
3. The `width` property should work for SVG rects. If it doesn't, use individual refs per bar.

**Reduced motion handling:**
