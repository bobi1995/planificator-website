## Objective

Create the before/after comparison section with a draggable slider that reveals business impact KPIs (manual scheduling vs. Planifactor). The section sits between Hero and FeatureHighlights on the home page. It uses `react-compare-slider` for the drag interaction, `motion/react` for scroll-triggered entrance animation, and includes full EN/BG translations. KPI cards show red/muted "before" metrics and green/brand "after" metrics in a 2x2 grid (mobile) or 4-column layout (desktop).

**File 1: `src/components/sections/MetricCard.tsx`**

This is a small presentational component. It does NOT need `"use client"` -- it receives props and renders JSX. It will be rendered inside a client component parent (ComparisonSlider), so it is implicitly a client component.

```typescript
import {cn} from '@/lib/utils';

interface MetricCardProps {
  value: string;
  label: string;
  icon: React.ReactNode;
  variant: 'before' | 'after';
}

export function MetricCard({value, label, icon, variant}: MetricCardProps) {
  return (
    <div className={cn(
      'rounded-lg border p-4 text-center',
      variant === 'before'
        ? 'bg-red-50 border-red-200'
        : 'bg-brand-50 border-brand-200'
    )}>
      <div className={cn(
        'mx-auto mb-2 [&>svg]:h-8 [&>svg]:w-8',
        variant === 'before' ? 'text-red-400' : 'text-brand-600'
      )}>
        {icon}
      </div>
      <div className={cn(
        'text-2xl md:text-3xl font-bold',
        variant === 'before' ? 'text-red-600' : 'text-brand-700'
      )}>
        {value}
      </div>
      <div className="text-sm text-muted-foreground mt-1">{label}</div>
    </div>
  );
}
```

Note on Tailwind v4 color classes: `bg-red-50`, `border-red-200`, `text-red-400`, `text-red-600` are built-in Tailwind colors (not custom). `bg-brand-50`, `border-brand-200`, `text-brand-600`, `text-brand-700` use the custom brand scale defined in `globals.css` via `@theme inline`. These should work because Tailwind v4 maps `--color-brand-*` variables to utility classes automatically.

**File 2: `src/components/sections/ComparisonSlider.tsx`**

This is a `"use client"` component. It renders the interactive before/after slider.

```typescript
'use client';

import {ReactCompareSlider, ReactCompareSliderHandle} from 'react-compare-slider';
import {m, LazyMotion, domAnimation, useReducedMotion} from 'motion/react';
import {Activity, Clock, Truck, Pause} from 'lucide-react';
import {MetricCard} from './MetricCard';
```

**Props interface:**
```typescript
interface Metric {
  label: string;
  before: string;
  after: string;
  icon: React.ReactNode;
}

interface ComparisonSliderProps {
  beforeLabel: string;
  afterLabel: string;
  metrics: Metric[];
}
```

**Component structure:**

The component renders two content panels -- "before" and "after" -- inside a `ReactCompareSlider`.

**Before content (left/top side):**
```tsx
<div className="p-6 md:p-8 h-full flex flex-col">
  <h3 className="text-label text-red-500 uppercase tracking-wider mb-6 font-semibold">
    {beforeLabel}
  </h3>
  <div className="grid grid-cols-2 gap-4 flex-1">
    {metrics.map((m) => (
      <MetricCard key={m.label} value={m.before} label={m.label} icon={m.icon} variant="before" />
    ))}
  </div>
</div>
```

**After content (right/bottom side):**
```tsx
<div className="p-6 md:p-8 h-full flex flex-col">
  <h3 className="text-label text-brand-600 uppercase tracking-wider mb-6 font-semibold">
    {afterLabel}
  </h3>
  <div className="grid grid-cols-2 gap-4 flex-1">
    {metrics.map((m) => (
      <MetricCard key={m.label} value={m.after} label={m.label} icon={m.icon} variant="after" />
    ))}
  </div>
</div>
```

**Slider rendering:**

Use `ReactCompareSlider` with `itemOne` (before) and `itemTwo` (after). For mobile (<768px), use `portrait` mode (horizontal divider, top/bottom). For desktop, use default landscape mode (vertical divider, left/right).

Render TWO `ReactCompareSlider` instances: one for desktop (hidden on mobile), one for mobile (hidden on desktop). Use Tailwind responsive classes `hidden md:block` and `block md:hidden`.

```tsx
{/* Desktop: vertical divider (landscape) */}
<div className="hidden md:block">
  <ReactCompareSlider
    itemOne={beforeContent}
    itemTwo={afterContent}
    position={50}
    handle={<ReactCompareSliderHandle buttonStyle={{backdropFilter: undefined, background: 'oklch(0.546 0.245 262.881)', border: 0, color: '#fff'}} />}
  />
</div>

{/* Mobile: horizontal divider (portrait) */}
<div className="block md:hidden">
  <ReactCompareSlider
    itemOne={beforeContent}
    itemTwo={afterContent}
    portrait
    position={50}
    handle={<ReactCompareSliderHandle portrait buttonStyle={{backdropFilter: undefined, background: 'oklch(0.546 0.245 262.881)', border: 0, color: '#fff'}} />}
  />
</div>
```

**Entrance animation:** Wrap the entire slider in a `LazyMotion` + `m.div` with `whileInView`:

```tsx
<LazyMotion features={domAnimation}>
  <m.div
    initial={shouldReduceMotion ? false : {opacity: 0, y: 30}}
    whileInView={{opacity: 1, y: 0}}
    viewport={{once: true, amount: 0.3}}
    transition={{duration: 0.6, ease: 'easeOut'}}
  >
    {/* Desktop and Mobile slider instances */}
  </m.div>
</LazyMotion>
```

If `shouldReduceMotion` is true, `initial` is `false` which means no initial hidden state -- content appears immediately with no animation. The slider remains interactive regardless.

**IMPORTANT: If `react-compare-slider` has import errors or runtime issues** (it IS a beta), fall back to a custom slider implementation. The custom approach uses:
- A container div with `position: relative`
- itemTwo rendered full-size as the background
- itemOne rendered with `position: absolute; inset: 0` and `clip-path: inset(0 ${100 - position}% 0 0)` for landscape or `inset(0 0 ${100 - position}% 0)` for portrait
- A drag handle div positioned at the clip boundary with `onPointerDown`/`onPointerMove`/`onPointerUp` handlers
- State for `position` (0-100) and `isDragging`

Only use this fallback if the beta package fails. Try the package first.

**File 3: `src/components/sections/ComparisonSection.tsx`**

This is an async server component (like FeatureHighlights, SocialProof, etc.) that wraps ComparisonSlider with translations and section structure.

```typescript
import {getTranslations} from 'next-intl/server';
import {Activity, Clock, Truck, Pause} from 'lucide-react';
import {ComparisonSlider} from './ComparisonSlider';
```

**Component:**
```typescript
export async function ComparisonSection() {
  const t = await getTranslations('Comparison');

  const metrics = [
    {
      label: t('resourceUtilization'),
      before: t('resourceUtilizationBefore'),
      after: t('resourceUtilizationAfter'),
      icon: <Activity />,
    },
    {
      label: t('onTimeDelivery'),
      before: t('onTimeDeliveryBefore'),
      after: t('onTimeDeliveryAfter'),
      icon: <Truck />,
    },
    {
      label: t('schedulingTime'),
      before: t('schedulingTimeBefore'),
      after: t('schedulingTimeAfter'),
      icon: <Clock />,
    },
    {
      label: t('idleTime'),
      before: t('idleTimeBefore'),
      after: t('idleTimeAfter'),
      icon: <Pause />,
    },
  ];

  return (
    <section className="py-20 px-4">
      <div className="max-w-5xl mx-auto">
        <div className="text-center mb-12">
          <h2 className="text-heading md:text-display">{t('sectionHeading')}</h2>
          <p className="mt-4 text-body-lg text-muted-foreground max-w-2xl mx-auto">
            {t('sectionSubtext')}
          </p>
        </div>

        <div className="min-h-[350px] md:min-h-[300px] rounded-xl border bg-card shadow-lg overflow-hidden">
          <ComparisonSlider
            beforeLabel={t('beforeLabel')}
            afterLabel={t('afterLabel')}
            metrics={metrics}
          />
        </div>
      </div>
    </section>
  );
}
```

Key details:
- `min-h-[350px] md:min-h-[300px]` reserves space to prevent CLS before client hydration
- `max-w-5xl` is narrower than the `max-w-7xl` used by Hero and Features -- this keeps the comparison compact and focused
- The section wrapper follows the same pattern as FeatureHighlights (py-20 px-4, centered heading + subtext)
- Lucide icons (`Activity`, `Truck`, `Clock`, `Pause`) are passed as JSX elements. They render as client components inside ComparisonSlider because ComparisonSlider has `"use client"`. This is fine -- server components can pass JSX as props to client components.

**IMPORTANT about passing JSX icons from server to client:** In Next.js App Router, you CAN pass React elements (like `<Activity />`) from server components as props to client components. These are serialized as React elements across the boundary. However, if this causes a serialization error, change the approach: pass icon NAME strings from the server component and resolve them to Lucide components inside the client component.
</instructions>
