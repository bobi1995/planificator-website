# Plan 01: Project Initialization & Toolchain

## Objective

Initialize the Next.js 16 App Router project with Tailwind CSS v4, shadcn/ui, and next-intl installed. Establish the component folder structure conventions. After this plan, the dev server runs and the toolchain is ready for i18n configuration.

Purpose: Every other plan in this phase (and every subsequent phase) depends on a working Next.js project with the correct dependencies installed.

Output: A running Next.js 16 project with Tailwind v4, shadcn/ui Button component, next-intl installed, and the component directory structure established.

## Context

@.planning/phases/01-foundation-i18n/01-RESEARCH.md — Contains exact package versions, installation commands, and anti-patterns.
@CLAUDE.md — Project overview and tech stack decisions.

**Key facts from research:**
- Next.js 16.1.6, React 19.2.4, next-intl 4.8.2, Tailwind CSS 4.1.18, shadcn/ui CLI 3.8.4, TypeScript 5.9.3
- Tailwind v4 uses CSS-first config (`@theme inline`), NOT `tailwind.config.ts`
- shadcn/ui style must be `new-york` (default is deprecated)
- Next.js 16 uses `proxy.ts` instead of `middleware.ts`
- Next.js 16 bundles Turbopack as default

<tasks>

<task id="1">
**Create Next.js 16 project and install all dependencies**

Run the following commands in sequence from the project root `C:\mypc\programming\Prefactor\marketing-site`:

```bash
# Step 1: Create Next.js 16 project (will scaffold into current directory)
npx create-next-app@16 . --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --turbopack
```

When `create-next-app` completes, verify:
- `package.json` exists and lists `next` as a dependency
- `src/app/layout.tsx` exists
- `src/app/globals.css` exists with Tailwind imports
- `tsconfig.json` exists with `@/*` path alias

Then initialize shadcn/ui:

```bash
# Step 2: Initialize shadcn/ui
# If prompted interactively, select: style=new-york, baseColor=neutral, cssVariables=yes
npx shadcn@latest init --style new-york --base-color neutral --css-variables --defaults
```

Verify `components.json` exists at project root with `"style": "new-york"`.

Then add the Button component and install next-intl:

```bash
# Step 3: Add shadcn Button component
npx shadcn@latest add button

# Step 4: Install next-intl
npm install next-intl
```

Verify:
- `src/components/ui/button.tsx` exists
- `package.json` includes `next-intl` in dependencies

**IMPORTANT anti-patterns:**
- Do NOT create `tailwind.config.ts` or `tailwind.config.js` -- Tailwind v4 uses CSS-first configuration
- If `create-next-app` creates a `middleware.ts`, delete it -- Next.js 16 uses `proxy.ts` (will be created in Plan 02)
- If prompted about style during shadcn init, always select `new-york`, never `default`

**Verify:** Run `npm run dev` and confirm the dev server starts on localhost:3000 without errors. Run `npm run build` and confirm it completes with zero errors.

**Done:** `package.json` lists `next` (v16+), `react` (v19+), `next-intl`, `tailwindcss` (v4+). `components.json` shows `style: "new-york"`. `src/components/ui/button.tsx` exists. Dev server starts cleanly.
</task>

<task id="2" depends_on="1">
**Establish component directory structure and clean up scaffolded files**

Create the component directory structure that enforces server/client component boundaries:

```bash
mkdir -p src/components/layout
mkdir -p src/components/sections
mkdir -p src/components/interactive
mkdir -p src/i18n
mkdir -p messages
```

These directories establish the convention:
- `src/components/ui/` — shadcn/ui generated components (already exists from shadcn init)
- `src/components/layout/` — Layout components like LanguageSwitcher, Header, Footer
- `src/components/sections/` — Server Components for page sections (Hero, Features, etc.)
- `src/components/interactive/` — Client Components with `"use client"` for stateful widgets
- `src/i18n/` — next-intl configuration files
- `messages/` — Translation JSON files

Clean up the default scaffolded page content. Edit `src/app/page.tsx` to be a minimal placeholder:

```typescript
export default function Home() {
  return (
    <main className="flex min-h-screen flex-col items-center justify-center p-24">
      <h1 className="text-4xl font-bold">Prefactor</h1>
      <p className="mt-4 text-lg text-muted-foreground">
        Marketing site — Phase 1 in progress
      </p>
    </main>
  );
}
```

Clean up `src/app/layout.tsx` to remove any default Next.js boilerplate fonts (Geist) since we will use Inter in Plan 02. Keep it minimal:

```typescript
import type { Metadata } from "next";
import "./globals.css";

export const metadata: Metadata = {
  title: "Prefactor",
  description: "Production Scheduling Software",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body className="antialiased">
        {children}
      </body>
    </html>
  );
}
```

After editing `globals.css`, ensure it contains the `--font-sans` variable pointing to Inter (this will be used when Inter is configured in Plan 02). Add this line inside the `@theme inline` block if not already present:

```css
--font-sans: var(--font-inter), ui-sans-serif, system-ui, sans-serif;
```

Delete any `middleware.ts` file if it was created by `create-next-app` (Next.js 16 uses `proxy.ts`):

```bash
rm -f src/middleware.ts middleware.ts
```

**Verify:** Run `npm run dev` -- dev server starts. Visit `http://localhost:3000` -- shows "Prefactor" heading. The following directories exist: `src/components/layout/`, `src/components/sections/`, `src/components/interactive/`, `src/i18n/`, `messages/`. No `middleware.ts` file exists anywhere. No `tailwind.config.ts` or `tailwind.config.js` exists.

**Done:** Component directory structure matches the convention from research. Default scaffolded content is cleaned up. Project is ready for i18n configuration.
</task>
