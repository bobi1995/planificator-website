# Plan 03: Language Switcher & Build Verification

## Objective

Create the LanguageSwitcher client component and integrate it into the locale layout. Verify the entire Phase 1 build succeeds end-to-end: locale routing, font preloading, language switching, and production build.

Purpose: The LanguageSwitcher is the user-facing proof that i18n works. This plan also serves as the Phase 1 gate -- if the build passes and locale switching works, the foundation is solid.

Output: A working LanguageSwitcher component in the layout that toggles between `/en` and `/bg`, plus a verified production build.

## Context

@.planning/phases/01-foundation-i18n/01-RESEARCH.md — Pattern 8 (LanguageSwitcher) and Pattern 6 (navigation helpers).

**Key facts:**
- LanguageSwitcher is a `"use client"` component (uses `useLocale`, `useRouter`, `usePathname`)
- Uses `@/i18n/navigation` for locale-aware `useRouter` and `usePathname`
- Uses `@/i18n/routing` for the list of available locales
- `router.replace(pathname, {locale: newLocale})` switches locale while preserving path

**File 1: Create `src/components/layout/LanguageSwitcher.tsx`**

```typescript
'use client';

import {useLocale, useTranslations} from 'next-intl';
import {usePathname, useRouter} from '@/i18n/navigation';
import {routing} from '@/i18n/routing';

export function LanguageSwitcher() {
  const locale = useLocale();
  const router = useRouter();
  const pathname = usePathname();
  const t = useTranslations('LanguageSwitcher');

  function switchLocale(newLocale: string) {
    router.replace(pathname, {locale: newLocale});
  }

  return (
    <div className="flex items-center gap-2">
      {routing.locales.map((loc) => (
        <button
          key={loc}
          onClick={() => switchLocale(loc)}
          disabled={loc === locale}
          className={`px-3 py-1.5 text-sm rounded-md transition-colors ${
            loc === locale
              ? 'bg-primary text-primary-foreground font-semibold cursor-default'
              : 'hover:bg-accent hover:text-accent-foreground'
          }`}
        >
          {t(loc)}
        </button>
      ))}
    </div>
  );
}
```

Key points:
- `"use client"` because it uses hooks (`useLocale`, `useRouter`, `usePathname`)
- Reads locale labels from translations (`LanguageSwitcher.en` = "English", `LanguageSwitcher.bg` = "Bulgarian")
- Current locale button is disabled and visually distinct (bold, primary colors)
- `router.replace` preserves the path when switching locale

**File 2: Update `src/app/[locale]/layout.tsx`** to include the LanguageSwitcher.

Read the existing locale layout first, then add the LanguageSwitcher import and render it in the body. The layout should become:

```typescript
import {NextIntlClientProvider, hasLocale} from 'next-intl';
import {notFound} from 'next/navigation';
import {routing} from '@/i18n/routing';
import {setRequestLocale} from 'next-intl/server';
import {Inter} from 'next/font/google';
import {LanguageSwitcher} from '@/components/layout/LanguageSwitcher';
import '../globals.css';

const inter = Inter({
  subsets: ['latin', 'cyrillic'],
  display: 'swap',
  variable: '--font-inter',
});

type Props = {
  children: React.ReactNode;
  params: Promise<{locale: string}>;
};

export function generateStaticParams() {
  return routing.locales.map((locale) => ({locale}));
}

export default async function LocaleLayout({children, params}: Props) {
  const {locale} = await params;
  if (!hasLocale(routing.locales, locale)) {
    notFound();
  }

  setRequestLocale(locale);

  return (
    <html lang={locale} className={inter.variable}>
      <body className="font-sans antialiased">
        <NextIntlClientProvider>
          <header className="flex justify-end p-4 border-b">
            <LanguageSwitcher />
          </header>
          <div className="flex-1">
            {children}
          </div>
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

Changes from Plan 02:
- Added `import {LanguageSwitcher}` from `@/components/layout/LanguageSwitcher`
- Added `<header>` with LanguageSwitcher before children
- Wrapped children in a `<div className="flex-1">` for layout structure
