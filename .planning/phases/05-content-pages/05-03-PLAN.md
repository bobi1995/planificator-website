## Objective

Build the Pricing page with 3 tier cards (Starter, Professional, Enterprise), a feature comparison table, and an FAQ accordion. Per STATE.md, no public prices are displayed -- all tiers have a "Contact for Pricing" CTA linking to `/contact`. The Professional tier is highlighted as "Most Popular" using a Badge. The comparison table shows 15-20 feature rows with checkmarks/dashes across tiers. The FAQ uses the shadcn Accordion component (client-side interactivity). The page ends with CTABanner.

Purpose: Manufacturing decision-makers need to understand tier differentiation to self-qualify before contacting sales. The comparison table helps them identify which tier matches their needs. The FAQ reduces friction by answering common objections.

Output: Fully built Pricing page with ~110 translation keys per locale, 4 new section components (PricingHero, PricingTierCard, PricingComparisonTable, PricingFAQ).

**File 3: `src/components/sections/pricing/PricingComparisonTable.tsx`**

Async server component. Renders the feature comparison matrix using the shadcn Table component.

```typescript
import {getTranslations} from 'next-intl/server';
import {
  Table,
  TableBody,
  TableCell,
  TableHead,
  TableHeader,
  TableRow,
} from '@/components/ui/table';
import {Check, X} from 'lucide-react';

const CATEGORIES = ['scheduling', 'resources', 'operations', 'support'] as const;

export async function PricingComparisonTable() {
  const t = await getTranslations('PricingPage.comparison');

  // Helper to render cell value
  function renderCell(value: unknown) {
    if (value === true) return <Check className="h-5 w-5 text-brand-600 mx-auto" />;
    if (value === false) return <X className="h-5 w-5 text-muted-foreground/40 mx-auto" />;
    // String value (e.g., "1 shift", "Unlimited")
    return <span className="text-sm font-medium">{String(value)}</span>;
  }

  return (
    <section className="py-20 px-4">
      <div className="max-w-5xl mx-auto">
        <h2 className="text-heading md:text-display text-center mb-12">
          {t('title')}
        </h2>
        <div className="overflow-x-auto rounded-lg border">
          <Table>
            <TableHeader>
              <TableRow>
                <TableHead className="w-[40%]"></TableHead>
                <TableHead className="text-center">{t('starter')}</TableHead>
                <TableHead className="text-center font-bold text-brand-600">{t('professional')}</TableHead>
                <TableHead className="text-center">{t('enterprise')}</TableHead>
              </TableRow>
            </TableHeader>
            <TableBody>
              {CATEGORIES.map((category) => {
                // Render category header row + feature rows
                const featureIndices = [0, 1, 2, 3, 4];
                return (
                  <>
                    <TableRow key={`${category}-header`} className="bg-muted/50">
                      <TableCell colSpan={4} className="font-semibold">
                        {t(`categories.${category}.title`)}
                      </TableCell>
                    </TableRow>
                    {featureIndices.map((i) => {
                      // Use raw message access for boolean values
                      // Since next-intl returns strings, we need to handle the
                      // comparison table values specially
                      const key = `categories.${category}.features.${i}`;
                      return (
                        <TableRow key={`${category}-${i}`}>
                          <TableCell>{t(`${key}.name`)}</TableCell>
                          <TableCell className="text-center">
                            {renderCell(t.raw(`${key}.starter`))}
                          </TableCell>
                          <TableCell className="text-center">
                            {renderCell(t.raw(`${key}.professional`))}
                          </TableCell>
                          <TableCell className="text-center">
                            {renderCell(t.raw(`${key}.enterprise`))}
                          </TableCell>
                        </TableRow>
                      );
                    })}
                  </>
                );
              })}
            </TableBody>
          </Table>
        </div>
      </div>
    </section>
  );
}
```

Key details:
- `t.raw()` is used to access the raw JSON value (boolean or string) rather than having next-intl try to format it
- Booleans render as Check (brand-blue) or X (muted) icons
- String values (like "1 shift", "Unlimited") render as text
- `overflow-x-auto` on the table container prevents mobile overflow
- Category header rows span all 4 columns with muted background
- Professional column header is highlighted with `text-brand-600 font-bold`

**IMPORTANT:** If `t.raw()` is not available in the version of next-intl being used, an alternative approach is to store the comparison data as a TypeScript constant (not in translations) and only translate the feature names and category titles. The feature availability (true/false/string) is not locale-dependent.
