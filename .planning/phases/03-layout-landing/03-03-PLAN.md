## Objective

Build the Hero section — the first thing visitors see when landing on the site. This includes a full-viewport section with the AI-focused headline, audience-focused subtext, two CTA buttons, and a static SVG Gantt chart mockup that communicates the product visually.

The Gantt mockup is an inline SVG React component (matching the Logo.tsx pattern) that will be replaced with an animated Framer Motion version in Phase 4. It must look polished and professional at all breakpoints, using brand colors from CSS variables.

## Tasks

<task id="1">
<title>Create static SVG Gantt chart mockup component</title>
<instructions>
Create `src/components/sections/GanttMockup.tsx` — a pure presentational React component that renders an inline SVG illustration of a Gantt chart.

**Design specifications:**
- ViewBox: `0 0 800 400` (wide aspect ratio, like a real Gantt chart)
- The SVG should depict a simplified but realistic production schedule with:
  - A light grid background with vertical day/time dividers
  - 5-6 horizontal resource rows (labeled as "Machine A", "Machine B", etc. or just generic row labels via subtle text)
  - Colorful task bars of varying lengths positioned across the timeline
  - Use brand color scale for task bars: `var(--color-brand-400)`, `var(--color-brand-500)`, `var(--color-brand-600)`, `var(--color-brand-700)` for variety
  - Rounded corners on bars (rx="4" or rx="6")
  - A subtle header row at the top showing time period labels (Mon, Tue, Wed, Thu, Fri — or just column lines)

**Color approach:**
- Background: use `var(--color-brand-50)` or a very light neutral
- Grid lines: `var(--color-brand-100)` or `var(--color-border)`
- Task bars: alternate between `var(--color-brand-400)`, `var(--color-brand-500)`, `var(--color-brand-600)`, `var(--color-brand-700)`
- Row labels (if included): `var(--color-muted-foreground)`
- Header text: `var(--color-foreground)`

**Important CSS variable usage in SVG:** Use `currentColor` or string references to CSS variables via `style` attribute on SVG elements. For example:
```tsx
<rect style={{fill: 'var(--color-brand-500)'}} ... />
```
This keeps the mockup in sync with the design system. Do NOT hardcode hex colors.

**Component interface:**
```typescript
interface GanttMockupProps {
  className?: string;
}

export function GanttMockup({className}: GanttMockupProps) {
  return (
    <svg
      viewBox="0 0 800 400"
      className={cn("w-full h-auto", className)}
      role="img"
      aria-hidden="true"
      xmlns="http://www.w3.org/2000/svg"
    >
      {/* Background */}
      {/* Grid lines */}
      {/* Row backgrounds (alternating subtle stripes) */}
      {/* Time header */}
      {/* Task bars */}
    </svg>
  );
}
```

Use `aria-hidden="true"` because the Hero section has a separate `aria-label` on the image wrapper. The SVG is decorative.

**Import:** `import {cn} from '@/lib/utils';`

**Sizing:** The SVG should be fully responsive via `viewBox` and `w-full h-auto`. No fixed width/height attributes on the SVG element itself.

**Bar layout example (approximate positions in the 800x400 viewBox):**
- Row height: ~60px, starting after a 40px header
- Header area: y=0 to y=40 (day labels)
- Row 1 (y=40-100): 2 bars — one from x=50-250 (brand-600), one from x=300-500 (brand-400)
- Row 2 (y=100-160): 1 long bar — x=100-550 (brand-500)
- Row 3 (y=160-220): 2 bars — x=200-400 (brand-700), x=450-650 (brand-400)
- Row 4 (y=220-280): 1 bar — x=50-350 (brand-600)
- Row 5 (y=280-340): 2 bars — x=150-300 (brand-500), x=350-700 (brand-700)
- Row 6 (y=340-400): 1 bar — x=250-600 (brand-400)

These are approximate — adjust for visual balance. The goal is to look like a real production schedule with staggered, overlapping tasks. Add subtle rounded-rect borders or shadows to make it feel professional.

**Add a subtle shadow/elevation effect** around the entire SVG by wrapping it in a div with `rounded-xl border bg-card shadow-lg overflow-hidden` in the Hero component (not in GanttMockup itself — keep the component pure SVG).
</instructions>

<task id="2">
<title>Create Hero section component</title>
<instructions>
Create `src/components/sections/Hero.tsx` as an async server component.

**Layout structure:**
```
<section> — full viewport height, centered content
  <div> — max-width container
    <div> — two-column on desktop (text left, Gantt right), stacked on mobile (text top, Gantt bottom)
      <div> — text content column
        <h1> — headline
        <p> — subtext
        <div> — CTA buttons row
      </div>
      <div> — visual column
        <div> — Gantt mockup wrapper with shadow/border
          <GanttMockup />
        </div>
      </div>
    </div>
  </div>
</section>
```

**Specific implementation:**

Section wrapper:
```tsx
<section className="min-h-dvh flex items-center py-20 px-4">
```
Use `min-h-dvh` (dynamic viewport height) to fill the viewport correctly on mobile browsers. If Tailwind v4 does not support `min-h-dvh` as a utility, fall back to `min-h-[100dvh]`. The `py-20` adds breathing room above/below so content doesn't feel jammed against header/bottom.

Container + grid:
```tsx
<div className="max-w-7xl mx-auto w-full">
  <div className="grid grid-cols-1 lg:grid-cols-2 gap-12 lg:gap-16 items-center">
```

**Text column (left on desktop, top on mobile):**
```tsx
<div className="text-center lg:text-left">
  <h1 className="text-heading md:text-display lg:text-hero">
    {t('headline')}
  </h1>
  <p className="mt-6 text-base md:text-body-lg text-muted-foreground max-w-xl mx-auto lg:mx-0">
    {t('subtext')}
  </p>
  <div className="mt-8 flex flex-col sm:flex-row gap-4 justify-center lg:justify-start">
    <Button size="lg" asChild>
      <Link href="/contact">{t('cta')}</Link>
    </Button>
    <Button variant="outline" size="lg" asChild>
      <Link href="/features">{t('ctaSecondary')}</Link>
    </Button>
  </div>
</div>
```

**Visual column (right on desktop, bottom on mobile):**
```tsx
<div className="flex justify-center lg:justify-end">
  <div className="w-full max-w-lg lg:max-w-none rounded-xl border bg-card shadow-lg overflow-hidden">
    <GanttMockup />
  </div>
</div>
```

The wrapper div adds the card-like elevation to the Gantt SVG. `max-w-lg` on mobile prevents it from stretching too wide, `lg:max-w-none` lets it fill the column on desktop.

**Imports:**
```typescript
import {getTranslations} from 'next-intl/server';
import {Link} from '@/i18n/navigation';
import {Button} from '@/components/ui/button';
import {GanttMockup} from '@/components/sections/GanttMockup';
```

**Export:** `export async function Hero() { ... }`
Use: `const t = await getTranslations('Hero');`

**Key details:**
- This is a SERVER component (async, uses `getTranslations`)
- The `Link` component is from `@/i18n/navigation` for auto locale handling
- CTA buttons use `asChild` to wrap Link (so the button is actually an anchor tag)
- The `Button` imports from `@/components/ui/button`
- On mobile (below lg), content stacks: text first, Gantt below. CTA buttons stack vertically below sm, side-by-side at sm+
