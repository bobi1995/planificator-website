## Objective

Install the two new shadcn/ui components needed by Phase 3 (Sheet for mobile navigation drawer, Avatar for testimonial placeholders), add all translation content for every new section (Hero, Features, SocialProof, CTABanner, Footer) to both en.json and bg.json, and create minimal stub pages for all navigation routes so no link in the site produces a 404.

This is the foundational plan -- every other plan in Phase 3 depends on these translations existing and these components being available.

## Tasks

<task id="1">
<title>Install shadcn/ui Sheet and Avatar components</title>
<instructions>
Run the shadcn CLI to install the Sheet and Avatar components:

```bash
npx shadcn@latest add sheet
npx shadcn@latest add avatar
```

After installation, verify the files exist at:
- `src/components/ui/sheet.tsx`
- `src/components/ui/avatar.tsx`

The Sheet component is built on Radix Dialog and will be used for the mobile navigation drawer in the Header. The Avatar component will be used for testimonial author photos in the SocialProof section.

Both components should import from `"radix-ui"` (the unified package already installed at ^1.4.3), matching the existing Button component pattern. If the shadcn CLI generates imports from individual `@radix-ui/react-*` packages instead, update them to use the unified `radix-ui` import.
</instructions>

<task id="3">
<title>Create stub pages for all navigation routes</title>
<instructions>
Create minimal stub pages for every route that will be linked from the Header and Footer navigation. Each stub page follows the same pattern: a server component that renders the page title and a "coming soon" message using translations from the `StubPage` namespace. Also use the page-specific namespace from `Navigation` for the title.

Create these 8 stub page files, all following the same pattern:

**Pattern for each stub page:**
```typescript
import {setRequestLocale, getTranslations} from 'next-intl/server';

type Props = {
  params: Promise<{locale: string}>;
};

export async function generateMetadata({params}: Props) {
  const {locale} = await params;
  const t = await getTranslations({locale, namespace: 'Navigation'});
  return {
    title: `${t('PAGE_KEY')} | Planifactor`,
  };
}

export default async function PAGE_NAMEPage({params}: Props) {
  const {locale} = await params;
  setRequestLocale(locale);
  const nav = await getTranslations({locale, namespace: 'Navigation'});
  const t = await getTranslations({locale, namespace: 'StubPage'});

  return (
    <div className="flex min-h-[60vh] flex-col items-center justify-center px-6 py-16">
      <h1 className="text-heading md:text-display mb-4">{nav('PAGE_KEY')}</h1>
      <p className="text-muted-foreground text-body-lg">{t('comingSoon')}</p>
      <p className="text-muted-foreground mt-2">{t('description')}</p>
    </div>
  );
}
```

**Files to create with their PAGE_KEY and route:**

1. `src/app/[locale]/features/page.tsx` — PAGE_KEY: `features`
2. `src/app/[locale]/use-cases/page.tsx` — PAGE_KEY: `useCases`
3. `src/app/[locale]/pricing/page.tsx` — PAGE_KEY: `pricing`
4. `src/app/[locale]/about/page.tsx` — PAGE_KEY: `about`
5. `src/app/[locale]/blog/page.tsx` — PAGE_KEY: `blog`
6. `src/app/[locale]/contact/page.tsx` — PAGE_KEY: `contact`
7. `src/app/[locale]/privacy/page.tsx` — PAGE_KEY: `privacy` (use Footer namespace, key `privacy`)
8. `src/app/[locale]/terms/page.tsx` — PAGE_KEY: `terms` (use Footer namespace, key `terms`)

For pages 7 and 8 (privacy, terms), use the `Footer` namespace instead of `Navigation` for the title since those keys are in the Footer namespace. Adapt the pattern accordingly.

Each stub page must include `setRequestLocale(locale)` for static rendering support and the `generateMetadata` export for proper page titles.
</instructions>
