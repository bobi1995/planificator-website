  <name>Task 1: CLS prevention and performance fixes</name>
  <files>
    src/components/interactive/CookieConsent.tsx
    src/components/interactive/CalendlyInline.tsx
  </files>
  <action>
    Audit and fix CLS (Cumulative Layout Shift) sources from dynamically rendered content:

    1. **CookieConsent banner** (`src/components/interactive/CookieConsent.tsx`):
       - The cookie consent banner appears at the bottom of the viewport on first visit
       - If it pushes content or appears without reserved space, it causes CLS
       - Ensure the banner uses `position: fixed` (bottom of viewport, overlay) so it does NOT push page content
       - If it already uses fixed positioning, verify it does not cause reflow
       - Check that the banner's show/hide transition does not trigger layout shifts (opacity/transform transitions are CLS-safe; height/margin are not)

    2. **CalendlyInline widget** (`src/components/interactive/CalendlyInline.tsx`):
       - The Calendly widget loads asynchronously via iframe
       - Without a reserved height, the iframe expanding causes CLS
       - Ensure the Calendly container has a `min-h-[700px]` (or appropriate value matching Calendly's default embed height) so space is reserved before the iframe loads
       - If the widget is hidden (consent not given), the fallback email content should also occupy similar space or use a graceful transition

    3. **General performance audit:**
       - Run `npm run build` and check for any large bundle warnings
       - Verify no `<img>` tags exist that should use `next/image` (search for `<img` in components -- the site is mostly SVG/CSS so this should be minimal)
       - Verify LazyMotion is used for motion animations (already done in Phase 4 -- just confirm)
       - Check that no client components import heavy server-only dependencies

    Only modify files if actual CLS or performance issues are found. If the components already handle these concerns correctly, document that in the summary.
  </action>
  <verify>
    - `npm run build` completes with zero errors and no large bundle warnings
    - CookieConsent uses fixed positioning (no content push)
    - CalendlyInline has reserved height for the iframe container
    - No raw `<img>` tags found in components (all should use next/image or be SVG)
  </verify>
  <done>

    - No performance anti-patterns found in components
    - Build produces zero errors
  </done>
