## Objective

Replace the Contact page stub with a full demo booking experience. The page features an inline Calendly scheduling widget as the primary conversion mechanism, flanked by contact information (email, phone, office hours). The Calendly widget is embedded via `react-calendly` if React 19 compatible, otherwise via a raw iframe fallback. All CTA buttons across the site continue linking to `/contact` (no server-to-client boundary changes).

Purpose: The Contact page is the terminal conversion point for every CTA on the site. Replacing the "Coming Soon" stub with an actual booking widget completes the visitor journey from interest to action.

Output: 1 npm package (react-calendly), 1 client component (CalendlyInline), 2 server section components (ContactHero, ContactInfo), 1 replaced page, ~20 new translation keys per locale.

**Step 2: Create CalendlyInline component at `src/components/interactive/CalendlyInline.tsx`**

This is a `'use client'` component that wraps the Calendly InlineWidget. It accepts a `url` prop (the Calendly scheduling link) and a `locale` prop (to pass locale to Calendly's URL). It also accepts a `consentGiven` prop to conditionally render based on cookie consent (this will be wired in Plan 07-03; for now, default to `true` so the widget always renders).

```typescript
'use client';

import {useTranslations} from 'next-intl';
import {InlineWidget} from 'react-calendly';
import {Mail} from 'lucide-react';

interface CalendlyInlineProps {
  url: string;
  locale?: string;
  consentGiven?: boolean;
}

export function CalendlyInline({url, locale, consentGiven = true}: CalendlyInlineProps) {
  const t = useTranslations('ContactPage');

  // Append locale to Calendly URL if provided (Calendly supports ?locale=xx)
  const calendlyUrl = locale && locale !== 'en'
    ? `${url}?locale=${locale}`
    : url;

  if (!consentGiven) {
    return (
      <div className="rounded-lg border border-dashed p-8 text-center bg-muted/30">
        <Mail className="h-10 w-10 text-muted-foreground mx-auto mb-4" />
        <p className="text-muted-foreground mb-2">{t('booking.consentRequired')}</p>
        <p className="text-sm text-muted-foreground">
          {t('booking.alternativeContact')}{' '}
          <a href="mailto:hello@planifactor.com" className="text-brand-600 underline">
            hello@planifactor.com
          </a>
        </p>
      </div>
    );
  }

  return (
    <div className="rounded-lg overflow-hidden border min-h-[700px]">
      <InlineWidget
        url={calendlyUrl}
        styles={{height: '700px', minWidth: '320px'}}
      />
    </div>
  );
}
```

Key implementation details:
- `'use client'` required because react-calendly components are interactive
- `consentGiven` prop defaults to `true` for now -- Plan 07-03 will wire this to cookie consent state
- When consent is not given, shows a placeholder with email fallback (site remains functional without cookies)
- Locale is appended as URL parameter to Calendly (Calendly's built-in localization)
- `min-h-[700px]` prevents layout shift while Calendly widget loads
- The fallback message uses `t('booking.consentRequired')` translation key

**Important note about react-calendly and React 19:** If the executor encounters TypeScript errors or runtime errors from `react-calendly` (most likely: `InlineWidget` type errors or SSR hydration issues), replace the `react-calendly` import with the raw iframe fallback:

```typescript
// Replace: import {InlineWidget} from 'react-calendly';
// With this inline component:
function InlineWidgetFallback({url, styles}: {url: string; styles: React.CSSProperties}) {
  return (
    <iframe
      src={url}
      style={styles}
      width="100%"
      frameBorder="0"
      title="Schedule a demo"
      loading="lazy"
    />
  );
}
// Then use InlineWidgetFallback instead of InlineWidget in the JSX
```

This provides identical functionality without the package dependency.

import {setRequestLocale, getTranslations} from 'next-intl/server';
import {ContactHero} from '@/components/sections/contact/ContactHero';
import {ContactInfo} from '@/components/sections/contact/ContactInfo';
import {CalendlyInline} from '@/components/interactive/CalendlyInline';

type Props = {
  params: Promise<{locale: string}>;
};

export async function generateMetadata({params}: Props) {
  const {locale} = await params;
  const t = await getTranslations({locale, namespace: 'ContactPage'});
  return {
    title: `${t('metaTitle')} | Planifactor`,
    description: t('metaDescription'),
  };
}

// Placeholder URL -- replace with real Calendly URL when available
const CALENDLY_URL = 'https://calendly.com/planifactor/demo';

export default async function ContactPage({params}: Props) {
  const {locale} = await params;
  setRequestLocale(locale);
  const t = await getTranslations({locale, namespace: 'ContactPage'});

  return (
    <>
      <ContactHero />

      <section className="py-16 px-4">
        <div className="max-w-7xl mx-auto">
          <div className="grid grid-cols-1 lg:grid-cols-3 gap-8">
            {/* Calendly widget -- takes 2/3 width on desktop */}
            <div className="lg:col-span-2">
              <h2 className="text-heading mb-2">{t('booking.title')}</h2>
              <p className="text-muted-foreground mb-6">{t('booking.description')}</p>
              <CalendlyInline url={CALENDLY_URL} locale={locale} />
            </div>

            {/* Contact info sidebar -- takes 1/3 width on desktop */}
            <div>
              <ContactInfo />
            </div>
          </div>
        </div>
      </section>
    </>
  );
}

Key details:

- Grid layout: 2/3 Calendly widget + 1/3 contact info sidebar on desktop; stacked on mobile
- `CALENDLY_URL` is a placeholder constant -- will be replaced with the real URL when the Calendly account is configured
- `locale` is passed to CalendlyInline for Calendly's built-in localization
